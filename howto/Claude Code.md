# Claude Code: инструкция по использованию

> Часть [[Руководство разработчика]]

## Установка

```bash
# Установка (требуется Node.js 18+)
npm install -g @anthropic-ai/claude-code

# Проверка
claude --version

# Авторизация
claude auth
```

Если работаешь из Zed или другого терминала, команда и workflow одинаковые.

## Базовые команды

```bash
# Запуск в текущей директории (Claude увидит весь проект)
claude

# Запуск с конкретной задачей
claude "добавь валидацию в FormRequest и обнови Vue-форму ошибки для страницы профиля"

# Продолжить предыдущую сессию
claude --resume

# Запуск в неинтерактивном режиме (для CI/CD)
claude --print "объясни что делает этот файл" < app/Http/Controllers/ProfileController.php
```

## Ключевые команды внутри сессии

| Команда | Что делает |
|---|---|
| `/clear` | Очистить контекст |
| `/compact` | Сжать контекст (когда окно забивается) |
| `/cost` | Показать потраченные токены |
| `/doctor` | Диагностика проблем с установкой |
| `Esc` | Отменить текущую генерацию |
| `Ctrl+C` дважды | Выход |

## CLAUDE.md и .cursor/rules

Claude Code читает файлы `CLAUDE.md` иерархически:

```
~/.claude/CLAUDE.md              ← глобальные персональные настройки
repo-root/CLAUDE.md              ← конвенции проекта (в git!)
repo-root/app/CLAUDE.md          ← backend/Laravel-правила (опционально)
repo-root/resources/CLAUDE.md    ← frontend/Vue-правила (опционально)
```

Параллельно учитываются `.cursor/rules/**` внутри проекта.

## Воркфлоу для типичных задач

### Реализация фичи по задаче

```
claude

> Реализуй задачу GP-456: "Добавить фильтры каталога растений"
  
  Требования:
  1. Product-план в docs/plans/product/gp-456.md уже согласован
  2. Создай dev-план в docs/plans/dev/gp-456.md и разбей на шаги
  3. Backend: FormRequest + Controller + Query scopes
  4. Frontend: Inertia page + Vue filters + Tailwind UI
  5. Не добавляй новые зависимости без согласования
  6. Перед кодом покажи план реализации
```

### Рефакторинг

```
claude

> У нас в resources/js/Pages/Plants дублируется логика фильтров.
  
  Задача: вынеси общий composable для фильтров и унифицируй query-параметры.
  Не меняй публичное поведение страницы.
  
  Покажи план перед тем как начнёшь менять файлы.
```

> [!tip] «Покажи план» — мощный приём
> Всегда просите план перед крупными изменениями. Это позволяет скорректировать подход до того, как агент начнёт менять файлы.

### Баг-фикс

```
claude

> Баг: при одновременной отправке двух форм профиль иногда сохраняется частично.
  
  Файлы:
  - app/Http/Controllers/ProfileController.php
  - app/Http/Requests/ProfileUpdateRequest.php
  - resources/js/Pages/Profile/Edit.vue
  
  Исправь с транзакцией и корректной серверной валидацией.
  Обнови клиентскую обработку ошибок Inertia.
  Перед правками покажи гипотезу причины.
```

## Работа через Quotio / агрегаторы

Если доступ к моделям организован через Quotio/агрегатор:
- проверяй, что используешь нужный профиль/endpoint перед сессией;
- не отправляй секреты и production-данные;
- сохраняй одинаковые правила качества независимо от провайдера модели.

## Агентный режим (мультифайловые операции)

Claude Code по умолчанию работает агентно: сам выбирает чтение/изменение файлов в рамках задачи. Для крупных задач задавай пошаговые критерии:

```
claude

> Приведи обработку ошибок в app/Http/Controllers к единому JSON-формату.
  
  Правила:
  1. Используй FormRequest для валидации
  2. Унифицируй исключения через Handler
  3. Не меняй внешние контракты API без согласования
  4. После каждого шага показывай короткий diff-результат
```

## Параллельные суб-агенты

Для действительно крупных задач Claude Code может запускать суб-агентов:

```
claude

> Нужно обновить все Go-модули до последних версий и пофиксить 
  все breaking changes. Используй параллельные воркеры для 
  каждого микросервиса в cmd/
```

## Интеграция с GitHub/GitLab через MCP

С настроенным MCP можно:

```
claude

> Посмотри открытые PR/MR, собери замечания ревью, обнови код и подготовь ответ
```

### Автоматическое создание MR

```
claude

> Реализуй фичу из задачи PROJ-789.
  Когда закончишь — создай MR в ветку develop 
  с описанием изменений.
```

## Лимиты и оптимизация

- Используй `/compact` когда контекст разрастается
- Для больших файлов: укажи конкретные строки или функции
- Если Claude «забывает» контекст — начни новую сессию с чётким промптом
- Следи за расходом через `/cost`

## Типичные ошибки

| Ошибка | Решение |
|---|---|
| Claude редактирует не тот файл | Укажи полный путь: `app/Http/Controllers/...` |
| Claude добавляет ненужные зависимости | Укажи: «используй только stdlib и существующие зависимости» |
| Claude перегенерирует весь файл | «Измени только функцию X, остальное не трогай» |
| Claude не учитывает наши конвенции | Проверь CLAUDE.md, добавь недостающее правило |
| Claude зацикливается на ошибке | `/clear`, переформулируй задачу |
