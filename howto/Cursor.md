# Cursor: инструкция по использованию

> Часть [[Руководство разработчика]]

## Установка и настройка

1. Скачать: [cursor.com](https://cursor.com)
2. Импортировать настройки из VS Code (Cursor предложит при первом запуске)
3. Установить расширения: PHP Intelephense, Vue Language Features (Volar), ESLint, Prettier, Tailwind CSS IntelliSense

## Ключевые горячие клавиши

| Комбинация | Действие |
|---|---|
| `Tab` | Принять автокомплит |
| `Esc` | Отклонить автокомплит |
| `Cmd+K` | Inline-редактирование (выдели код → опиши изменение) |
| `Cmd+L` | Открыть AI-чат |
| `Cmd+I` | Composer Mode (мультифайловое редактирование) |
| `Cmd+Shift+L` | Добавить выделенный код в чат как контекст |

## Три режима работы

### 1. Автокомплит (Tab)

Работает автоматически при наборе. Cursor предсказывает следующие строки на основе контекста файла и проекта.

Когда полезен:
- Бойлерплейт (контроллеры, FormRequest, ресурсы, composables)
- Повторяющиеся паттерны в Laravel/Vue коде
- Быстрые UI-итерации по Tailwind
- Заполнение типовых веток валидации/обработки ошибок

Когда отклонять (`Esc`):
- Предложение не учитывает бизнес-логику
- Генерирует устаревший API
- Предлагает `any` вместо конкретного типа во Vue/TS

### 2. Inline Edit (Cmd+K)

Выделяешь блок кода → нажимаешь `Cmd+K` → описываешь изменение.

Примеры:
```
"добавь обработку ошибки и логирование"
"вынеси правила в FormRequest"
"добавь тёмную тему через dark: классы"
"приведи страницу к паттерну существующих Inertia Pages"
```

Показывает diff перед применением — всегда проверяй.

### 3. Composer Mode (Cmd+I)

Мультифайловое редактирование с превью всех изменений.

Примеры задач для Composer:
```
"создай CRUD для коллекций растений:
 - маршруты в routes/web.php
 - Controller + FormRequest + Policy
 - Inertia-страницы в resources/js/Pages/Collections
 - Tailwind-формы и таблицу
 Следуй паттернам существующих страниц"
```

> [!tip] Composer показывает diff для каждого файла
> Просмотри все изменения перед Apply. Можно принять/отклонить каждый файл отдельно.

## .cursorrules — конвенции проекта

Файл `.cursorrules` в корне репозитория определяет правила для Cursor. Работает аналогично CLAUDE.md но проще.

У нас `.cursorrules` уже настроен в репо. Если нужны персональные правила — добавь в `~/.cursor/rules/`.

## Контекст — ключ к качеству

### Добавление контекста в чат

- `@file` — добавить файл в контекст
- `@folder` — добавить папку
- `@codebase` — поиск по всему проекту
- `@web` — поиск в интернете
- `@docs` — поиск по документации

### Типичный промпт с контекстом

```
@app/Http/Controllers/PlantController.php @resources/js/Pages/Plants/Index.vue

Добавь фильтрацию по освещённости и уровню полива:
1. Backend: валидация query-параметров и фильтры в query builder
2. Frontend: селекты + синхронизация с URL через Inertia
3. UI: Tailwind, dark mode и пустое состояние

Следуй паттерну существующих фильтров на странице коллекций
```

## Воркфлоу для ежедневной работы

### Утренний ритуал
1. `git pull`
2. Открыть задачу в трекере
3. Прочитать спеку / описание задачи
4. **Продумать архитектуру самостоятельно** (какие файлы, какие изменения)
5. Начать с Composer Mode для скаффолдинга
6. Переключиться на Tab-автокомплит для наполнения

### Для Laravel-разработчика

```
Типичная сессия:
1. Cmd+I: "создай FormRequest + Controller action по паттерну @app/Http/Controllers"
2. Tab: автокомплит дополняет rules/messages/authorize
3. Cmd+K: "добавь eager loading и pagination"
4. Cmd+K: "обнови Inertia props в соответствующей странице Vue"
5. Terminal: php artisan test --filter=<target>
```

### Для Vue-разработчика

```
Типичная сессия:
1. Cmd+I: "создай Vue-компонент фильтров по аналогии с @resources/js/Components/..."
2. Tab: автокомплит для props/emits и computed
3. Cmd+K: "добавь loading/skeleton и empty state"
4. Cmd+K: "приведи Tailwind-классы к проектному стилю"
5. Terminal: eslint + выбранные тесты при необходимости
```

## Интеграция с GitLab

Через MCP-сервер GitLab (настраивается в Settings → MCP):
- Cursor видит issues и MR
- Может создавать ветки и MR
- Может читать комментарии ревьюеров

## Чего НЕ делать в Cursor

- **Не принимай Tab автокомплит не глядя** — читай каждое предложение
- **Не используй Composer для >5 файлов одновременно** — качество падает
- **Не полагайся на @web для актуальной документации** — проверяй первоисточник
- **Не используй для security-critical кода** без последующего ручного ревью
