# Работа с AI-агентами: руководство разработчика

> Базовый документ по работе с AI-агентами для продуктовой команды Laravel/Vue.

## Почему это важно

AI ускоряет поставку фич только при дисциплине: чёткий контекст, небольшие шаги, ручная проверка и контроль архитектуры. Без этого он ускоряет только генерацию техдолга.

## Наш стек AI-инструментов

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **Claude Code** | Терминальный агент, мультифайловые правки, git/PR операции | Большие задачи, рефакторинг, сквозные изменения в backend+frontend |
| **Cursor** | IDE-ассистент, автокомплит, быстрые локальные изменения | Ежедневная работа, изменения в 1-5 файлах |
| **Figma Make** | Генерация React-черновика из дизайна | Быстрый старт UI перед адаптацией в Laravel Inertia + Vue |
| **Quotio / агрегаторы** | Доступ к моделям и прокси-инфраструктуре подписок | Централизованный доступ к LLM и масштабирование usage |

## Операционная модель команды

1. Product пишет требования и acceptance criteria в Markdown (`docs/plans/product/*.md`)
2. Dev пишет технический план и разбиение на шаги в Markdown (`docs/plans/dev/*.md`)
3. Реализация идёт итерациями с проверяемыми результатами
4. Для UI: Figma Make -> React draft -> адаптация в проектный стек (Inertia/Vue/Tailwind)

## Быстрый старт

```
1. Прочитай «Золотые правила» и «Безопасность»
2. Настрой Claude Code и Cursor
3. Изучи CLAUDE.md + .cursor/rules в рабочем репозитории
4. Перед реализацией создай/обнови product/dev планы в MD
5. Начни с малых задач и повышай сложность постепенно
6. Делись эффективными промптами и анти-паттернами внутри команды
```
