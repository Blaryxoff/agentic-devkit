# Ревью AI-сгенерированного кода

> Часть [[Руководство разработчика]]

## Принцип: AI-код — это черновик, не результат

AI-сгенерированный код нуждается в **более внимательном ревью**, чем написанный человеком. Причина: человек обычно понимает контекст и ограничения, AI — нет.

## На что обращать внимание при ревью

### 1. Идиоматичность Laravel/PHP

AI часто пишет Laravel-код как "обычный PHP без фреймворка". Красные флаги:

```php
// ❌ Валидация и бизнес-правила в контроллере
public function store(Request $request)
{
    $request->validate(['name' => 'required']);
    // ...
}

// ✅ FormRequest + тонкий контроллер
public function store(StorePlantRequest $request): RedirectResponse
{
    // ...
}
```

```php
// ❌ Сырой SQL из строк
$rows = DB::select("select * from plants where name like '%{$term}%'");

// ✅ Query Builder / Eloquent
$rows = Plant::query()->where('name', 'like', "%{$term}%")->get();
```

### 2. Ненужные абстракции

AI обожает создавать лишние сервис-слои и обёртки там, где хватит существующих Laravel-паттернов.

Проверяй, что:
- нет дублирующих "Manager/Helper/Utils" классов без ценности;
- используются FormRequest, Policy, Resource, Eloquent scopes там, где уместно;
- нет копипасты между Vue-страницами (лучше вынести в composable/component).

### 3. Обработка ошибок

Частая проблема и в backend, и в frontend. Проверяйте:

- [ ] Исключения конвертируются в понятные ответы/флеши
- [ ] Validation errors корректно отображаются на Vue-форме
- [ ] Нет "тихих" catch-блоков без логирования и обработки

### 4. Данные и запросы

- [ ] Нет N+1 в индексных страницах
- [ ] Для тяжёлых данных есть пагинация/ленивая загрузка
- [ ] Запросы в контроллере не перегружены лишней логикой

### 5. Зависимости

> [!danger] AI может предложить зависимость, которая:
> - Заброшена (нет коммитов >1 года)
> - Имеет уязвимости
> - Является malware (особенно npm)
> - Дублирует функциональность стандартной библиотеки

Перед принятием новой зависимости:
```bash
# PHP / JS
composer audit
npm audit                                 # уязвимости?
npx socket-npm <package>                  # supply chain проверка
```

### 6. Тесты

AI-сгенерированные тесты часто покрывают только happy path.
Проверяй наличие failure/edge сценариев и соответствие реальному поведению, а не внутренней реализации.

## Процесс ревью AI-кода в команде

### Автор MR

1. Сгенерировал код -> прогнал целевые проверки проекта (форматирование, линтеры, релевантные тесты)
2. Прочитал **каждую строку**, убедился что понимает
3. Разбил на логические коммиты
4. В описании MR указал: `🤖 AI-assisted: Claude Code / Cursor`
5. Отметил зоны повышенного внимания (см. [[Безопасность]])
6. Создал MR → прошёл автоматический ревью PR-Agent

### Ревьюер

1. Проверяет автоматические замечания и CI
2. Фокус на: Laravel-паттерны, обработку ошибок, безопасность, UX регрессии
3. **Не торопится** — AI-код выглядит убедительно, но может содержать тонкие баги
4. Проверяет новые зависимости
5. Проверяет критические сценарии вручную, если автотестов недостаточно

### Правило двух ревьюеров

Для кода из 🔴 красной зоны (см. [[Безопасность]]) — **обязательно два ревьюера**, один из которых — сеньор или лид.
