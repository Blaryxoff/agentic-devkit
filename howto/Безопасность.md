# Безопасность при работе с AI-агентами

> Часть [[Руководство разработчика]]

> [!danger] Ошибка безопасности в веб-продукте = утечки, компрометация аккаунтов и потеря доверия. AI-агенты не понимают реальную модель угроз вашего бизнеса.

## Что НЕЛЬЗЯ отправлять AI-агентам

### Абсолютный запрет
- Приватные ключи, seed-фразы, мнемоники
- Production API-ключи и токены
- Пароли, токены доступа, JWT-секреты
- Персональные данные пользователей (документы, телефоны, адреса, e-mail)
- Содержимое `.env` файлов с production-секретами

### Ограниченно (только с замаскированными данными)
- Конфигурации инфраструктуры (замените реальные IP, домены)
- Логи с пользовательскими данными (анонимизируйте)
- SQL-дампы (используйте фейковые данные)

### Допустимо
- Исходный код (без секретов)
- Тестовые конфигурации
- Публичная документация
- Ошибки компиляции и стек-трейсы (без PII)

## Зоны повышенного внимания

### 🔴 Красная зона — AI генерирует, человек проверяет КАЖДУЮ строку

Код, связанный с:

**Криптография и ключи**
- Генерация, хранение, ротация ключей
- Шифрование/дешифрование
- Хеширование паролей (bcrypt параметры, salt)
- TLS/mTLS конфигурация

**Аутентификация и доступ**
- Login/logout/session lifecycle
- Password reset, e-mail verification
- Gates/Policies/permissions
- Account takeover protections

**Авторизация**
- Проверка прав доступа
- JWT валидация и генерация
- Rate limiting
- RBAC/ABAC правила

**Пользовательский ввод**
- Валидация и санитизация
- SQL-запросы (инъекции)
- Сериализация/десериализация (RCE через unsafe unmarshal)

### 🟡 Жёлтая зона — AI генерирует, обязателен тщательный ревью

- Работа с базой данных (миграции, индексы, транзакции)
- Очереди, jobs, retries и idempotency
- Сетевые вызовы (таймауты, ретраи, circuit breaker)
- Конфигурация middleware (CORS, CSP, security headers)
- Docker/K8s конфигурации

### 🟢 Зелёная зона — AI генерирует свободно

- Юнит-тесты
- Документация и комментарии
- CLI-утилиты и скрипты
- Бойлерплейт-код (CRUD, маппинг)
- Прототипы и PoC
- Рефакторинг без изменения логики

## Правила для Laravel/PHP-кода

### Валидация и авторизация
Проверяйте, что AI не делает inline-валидацию в контроллере без причины. Предпочтительны FormRequest + Policy/Gate.

### SQL и Eloquent
Проверяйте:
- нет сырых SQL-строк из пользовательского ввода;
- нет N+1 (используется `with()` при необходимости);
- массовое присвоение защищено (`$fillable`/`$guarded`).

### Секреты и конфигурация
Проверяйте:
- нет `env()` вне `config/*.php`;
- нет секретов в логах, исключениях и фронтенд-props.

## Правила для Vue/TypeScript-кода

### Типизация
AI часто использует `any` или неточные типы. Проверяйте:
```typescript
// ❌ AI любит
const data: any = await fetchBalance()

// ✅ Должно быть
interface BalanceResponse {
  available: bigint
  locked: bigint
  currency: string
}
const data: BalanceResponse = await fetchBalance()
```

### XSS и рендеринг
Проверяйте:
- нет небезопасного HTML-рендеринга без санитизации;
- нет вывода сырого пользовательского контента.

## Чеклист безопасности перед MR

Перед созданием MR с AI-сгенерированным кодом, проверь:

- [ ] Код не содержит захардкоженных секретов
- [ ] Нет `any` типов в TypeScript (или они обоснованы)
- [ ] Валидация вынесена в FormRequest, авторизация в Policy/Gate
- [ ] SQL-запросы параметризованы (нет конкатенации строк)
- [ ] Пользовательский ввод валидируется
- [ ] Логирование не содержит PII/секретов
- [ ] Зависимости проверены (нет подозрительных новых пакетов)
- [ ] Форматирование и линтеры проходят на изменённых файлах
