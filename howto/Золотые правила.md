# Золотые правила работы с AI-агентами

> Часть [[Руководство разработчика]]

## Правило №1: Ты — архитектор, агент — исполнитель

AI-агент **не принимает архитектурных решений**. Ты определяешь:
- Какие пакеты и абстракции использовать
- Как организовать код
- Какие паттерны применять
- Какие trade-off'ы делать

Агент реализует твоё решение. Если ты не можешь объяснить агенту, что нужно сделать — значит ты сам ещё не понял задачу. Сначала подумай, потом делегируй.

## Правило №2: Никогда не мерджи то, что не понимаешь

> [!danger] Абсолютное правило
> Если ты не можешь объяснить каждую строку сгенерированного кода на ревью — **ты не имеешь права его мерджить**.

AI генерирует правдоподобный, но не всегда правильный код. Типичные проблемы:
- Устаревшие API (агент обучен на старых данных)
- Неидиоматичный Laravel/Vue код
- Игнорирование конвенций проекта
- «Галлюцинации» — несуществующие функции, пакеты, методы
- Скрытые баги в edge cases

## Правило №3: Контекст — это всё

Качество выхода агента прямо пропорционально качеству входа. Плохой промпт → плохой код.

### Плохо:
```
Напиши обработку ошибок
```

### Хорошо:
```
В app/Http/Controllers/PlantController.php метод index не валидирует query-параметры.
Нужно:
1. Создать PlantFilterRequest с rules/messages/authorize
2. Применить фильтры через scope в модели Plant
3. Передавать фильтры в Inertia::render
4. Обновить resources/js/Pages/Plants/Index.vue для отображения ошибок
Следуй конвенциям из CLAUDE.md
```

### Формула хорошего промпта:
```
ЧТО (конкретная задача)
+ ГДЕ (файлы, пакеты, модули)
+ КАК (паттерны, конвенции, ограничения)
+ КОНТЕКСТ (зачем это нужно, бизнес-логика)
```

## Правило №4: Итерируй, не перегенерируй

Если результат на 80% правильный — **дорабатывай**, а не перегенерируй с нуля. Каждая перегенерация:
- Теряет контекст предыдущих исправлений
- Может сломать то, что работало
- Тратит токены и время

Вместо: «Переделай всё»
Говори: «В строке 42 замени X на Y, в строке 58 добавь проверку Z»

## Правило №5: Один промпт — одна задача

Не просите агента сделать всё сразу. Декомпозируйте:

### Плохо:
```
Сделай личный кабинет пользователя целиком
```

### Хорошо:
```
Шаг 1: Product-план и acceptance criteria в docs/plans/product/
Шаг 2: Dev-план и техническая декомпозиция в docs/plans/dev/
Шаг 3: Backend (маршрут + контроллер + FormRequest + policy)
Шаг 4: Frontend (Inertia Page + компоненты + Tailwind)
Шаг 5: Финальная ручная проверка и ревью
...
```

## Правило №6: Верифицируй через запуск

> [!warning] Всегда после генерации
> ```bash
> php artisan test --filter=<target>   # если запуск тестов разрешён в процессе
> vendor/bin/pint --dirty              # форматирование PHP
> eslint <target-files>                # фронтенд-проверки
> ```

Никогда не доверяйте агенту на слово, что «код работает».

## Правило №7: Git-гигиена

- **Один коммит = одно логическое изменение**, даже если агент сгенерировал всё разом
- Пишите осмысленные коммит-сообщения, а не «AI-generated code»
- Используйте **draft MR** для AI-сгенерированных изменений
- В описании MR указывайте: «Создано с помощью Claude Code / Cursor, проверено вручную»
- Не коммитьте промпт-историю и логи агентов

## Правило №8: Учись на выходе агента

Каждая генерация — возможность учиться:
- Агент использовал паттерн, который ты не знал? Разберись.
- Агент сделал иначе, чем ты бы сделал? Сравни подходы.
- Агент ошибся? Запомни тип ошибки, чтобы ловить в будущем.

Делись находками в `#ai-tips`.

## Правило №9: Не используй AI для security-critical кода

Подробности в [[Безопасность]], но ключевое:

> [!danger] Запрещено делегировать AI без тщательной ручной верификации:
> - Криптографию и работу с ключами
> - Авторизацию и контроль доступа
> - Санитизацию пользовательского ввода
> - Бизнес-критичные расчёты и биллинг-логику

## Правило №10: Меряй свою продуктивность честно

Задай себе вопросы в конце недели:
- Сколько MR я замерджил? Больше, чем без AI?
- Сколько багов вернулось с QA? Больше, чем обычно?
- Сколько времени я потратил на исправление AI-кода vs написание своего?
- Понимаю ли я весь код, который попал в main за эту неделю?

Если AI не делает тебя продуктивнее — измени подход, а не увеличивай usage.
